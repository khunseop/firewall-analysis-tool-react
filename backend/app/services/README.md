# Policy Indexer (정책 인덱서)

정책 인덱서는 방화벽 정책의 소스(Source), 목적지(Destination), 서비스(Service) 정보를 분석하여 검색 및 분석이 용이한 형태의 데이터베이스 인덱스로 변환하는 핵심 서비스입니다.

## 아키텍처 및 설계 원칙

- **In-Memory 처리**: 모든 그룹 확장 및 분석 로직은 메모리 내에서 수행되어 DB I/O 부하를 최소화합니다.
- **고성능 자료형**: Python의 `Set`과 `Dict`를 활용하여 합집합 연산 및 멤버십 체크를 $O(1)$ 또는 $O(n)$ 시간에 처리합니다.
- **멱등성(Idempotency)**: 동일한 정책에 대해 실행 시 기존 인덱스를 삭제하고 새로 생성하여 데이터 일관성을 유지합니다.

## 데이터 흐름 (Data Flow)

1.  **데이터 로드**: 지정된 장치(Device)의 모든 네트워크 객체, 서비스 객체, 그룹 및 정책 정보를 단일 쿼리 세트로 로드합니다.
2.  **객체 리졸빙 (Resolving)**: `Resolver` 클래스가 중첩된 그룹 구조를 분석하여 각 객체 이름이 가지는 최종 값(IP 범위 또는 포트)을 매핑합니다.
3.  **정책 분석**: 각 정책의 멤버(문자열)를 리졸버 결과와 매핑하여 모든 포함 객체를 찾아냅니다.
4.  **데이터 압축**: IP 범위 병합 알고리즘을 통해 파편화된 IP 주소들을 최소한의 연속된 범위로 합칩니다.
5.  **일괄 저장 (Bulk Insert)**: 생성된 로우(Row) 데이터를 `bulk_insert_mappings`를 통해 DB에 고속 저장합니다.

## 주요 알고리즘

### 1. 재귀적 그룹 확장 (Recursive Group Expansion)
- **목적**: 중첩 그룹(그룹 내 그룹)을 최하위 기본 객체 리스트로 전개합니다.
- **구현**: 깊이 우선 탐색(DFS) 알고리즘을 사용하며, 메모이제이션(캐싱)을 통해 동일 그룹의 반복 계산을 방지합니다.
- **순환 참조 방지**: `visited` 세트를 통해 그룹 간 순환 참조가 발생하더라도 무한 루프에 빠지지 않도록 설계되었습니다.

### 2. IP 범위 병합 (IP Range Merging)
- **목적**: 인덱스 테이블의 크기를 줄이고 검색 효율을 높이기 위해 중복되거나 인접한 IP 범위를 통합합니다.
- **알고리즘**: Greedy Interval Merging
    1. 모든 IP/CIDR/범위를 숫자형 `(start, end)` 튜플로 변환합니다.
    2. 시작점(`start`)을 기준으로 오름차순 정렬합니다.
    3. 인접하거나 겹치는 범위(`next_start <= current_end + 1`)를 하나로 합칩니다.

### 3. 빈 그룹 처리 (Empty Group Handling)
- 멤버가 없는 그룹도 정책에 사용될 수 있으므로, 이를 검색 결과에서 누락시키지 않기 위해 `__GROUP__:` 접두사를 가진 특수 토큰으로 인덱싱합니다.

## 성능 최적화 전략

- **N+1 문제 해결**: `crud` 기능을 사용하여 필요한 모든 데이터를 초기에 대량 로드합니다.
- **SQLite 제약 대응**: SQLite의 변수 제한(999개)을 고려하여 `delete` 및 `insert` 작업을 적절한 청크(Chunk) 단위로 분할 처리합니다.
- **캐싱**: 포트 파싱 및 그룹 분석 결과를 캐싱하여 중복 연산을 제거합니다.
